# 通关法（Method: TongGuan）

## 来源

《滴天髓》— 通关论。蒋文正《论命琐记》特别强调。

## 功能定位

蒋文正批断流程 **Step 4**。检测命局中两种五行对峙（相克且势均力敌），若存在则取通关元素化解。

这是当前系统中**缺失**的方法。现有代码中仅有空注释 `// ── 5. 通关法 ──`。

## 输入

```typescript
interface TongGuanInput {
  pillars: Record<string, Pillar>;     // 四柱
  elementScores: Record<string, number>; // 五行力量分布（来自扶抑法输出）
}
```

## 输出

```typescript
type TongGuanOutput = {
  yongShen: string;  // 通关用神（五行元素）
  reason: string;    // 判断依据
} | null;            // null = 无对峙，无需通关
```

## 判断逻辑

### 1. 找出对峙双方

遍历所有五行相克配对：

| 克方 | 被克方 | 通关元素 |
|------|--------|----------|
| 金 | 木 | 水（金生水，水生木） |
| 木 | 土 | 火（木生火，火生土） |
| 土 | 水 | 金（土生金，金生水） |
| 水 | 火 | 木（水生木，木生火） |
| 火 | 金 | 土（火生土，土生金） |

### 2. 判断对峙条件

对于每对克方（A）和被克方（B）：
- **力量接近**：`|score[A] - score[B]| ≤ max(score[A], score[B]) × 0.5`
  - 即两者得分差距不超过高者的 50%
- **双方都有实质力量**：`score[A] >= 5` 且 `score[B] >= 5`
- **造成流通断裂**：在 木→火→土→金→水→木 的循环中，A 和 B 之间的通关元素（C）缺失或得分 < 3

### 3. 确定通关元素

取 A 和 B 之间的五行循环中介元素 C：
- 五行循环: 木 → 火 → 土 → 金 → 水 → 木
- 在 A→C→B 中，A 生 C，C 生 B

### 4. 返回

- 存在对峙 → `{ yongShen: C, reason: "A克B对峙，取C通关，A生C生B，化解冲突。" }`
- 多组对峙 → 取力量最强的对峙组
- 无对峙 → `null`

## 示例

**蔡建华** （庚金日主，力量分布: 水21 金15 土6 木4 火0）：
- 本局无实质性对峙（火0，水虽旺但克火无对象）
- 通关法返回 null → 扶抑法主力决断

**假想案例** （甲木日主，力量分布: 金18 木14 水3 火5 土0）：
- 金（18）克木（14）：|18-14| = 4，max × 0.5 = 9，4 ≤ 9 ✓
- 对峙成立 → 通关元素 = 水（金生水，水生木）
- 用神 = 水

## 古典依据

《滴天髓》通关论：
> 两神相峙，取通关为要。

蒋文正《论命琐记》：
> 金木交战，水为通关；水火交战，木为通关。通关即生机。

## 使用位置

在 `yongshen.ts` 的 `determineYongShen()` 中，第四步调用。在「扶抑法」之后、「病药法」之前。

通关法是否被采用取决于优先级：
- 若通关法返回非 null → 最终用神 = 通关元素
- 若通关法返回 null → 跳过，继续病药法
